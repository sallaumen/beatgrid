## Decisions

- **Build order:** Biblioteca first вҶ’ then Painel / Central de RevisГЈo / Detalhe.
- **Undo:** a dedicated **`operations`** table (from/to/kind/timestamp/status/batch) for
  persistent undo of applied disk operations.
- **Applying to disk:** apply real disk changes incrementally as we build (high-confidence
  batches, with undo). Renames/moves are safe before the library is indexed by Serato.
